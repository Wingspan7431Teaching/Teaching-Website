Constants File

Earlier, in the lesson on Command-Based Programming, constants were mentioned. A constant is a variable that does not change while the program is running. Common constants include motor IDs, wheel sizes, chassis size, default speed, and many other such values.

Putting all the constants in a specific file(s) is important so that you don’t have to go through the code and track down the variable to change it. It also makes it easier to reuse code. If you have all the physical constants in one place, it’s easy to change the numbers and use the same code for a different robot.

Some teams have one constants file for their entire robot. Wingspan uses one constants file per subsystem. We’ve found that it’s easier to find and organize values this way. However, the most important thing is to follow one convention for the whole team.

Variables in the constants file should always be declared as public static final. You can read about the final keyword here.
