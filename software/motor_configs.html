Motor Configuration

There’s a lot of different properties that we might want to edit on the motor. For example, motors have different behaviors for when they’re not moving. One behavior is brake where the motor actively resists movement. Another is coast where the motor can move relatively easily when force is applied. This is one property we can set using motor configurations. A motor configuration is an object that contains a bunch of properties that we can then use to set for a motor at once.

Let’s make a configuration for that SparkFlex motor from the last lesson. Since it’s a SparkFlex motor we’ll use the SparkFlexConfig constructor (if it were a SparkMax motor you’d use SparkMaxConfig). Create a SparkFlexConfig like so:

    SparkFlexConfig sparkFlexConfig = new SparkFlexConfig();

Now, to edit this configuration we’ll use one of the many methods this class offers.

    sparkFlexConfig.idleMode(IdleMode.kBrake);

This will set the behavior of the motor when it’s in idle mode (not moving) to break.

Another property that should always be set on motors controllers is a current limit. This is the max rate at which current may be sent to a motor. If we don’t set a current limit, too much power can be sent to the motor, which breaks it. We can set the current limit with .smartCurrentLimit(int stallLimit) like so:

    sparkFlexConfig

     .idleMode(IdleMode.kBrake)

     .smartCurrentLimit(60); //set current limit to 60 amps

Now that we have a configuration with the properties we want, how do we apply it to the motor controller? For this we’ll use the configure() method on the motor controller.

The configure() method has three main parameters: the config object, a boolean for whether the previous settings on the motor controllers should be reset, and a boolean for whether the config should persist in the motor controller’s memory or not.

  • boolean reset
    • When a motor controller gets configured, its settings are saved on the controller. Some settings will already be on there that you’re not altering with code. If reset is false, whatever was on there before will stay. If reset is true, the motor controller will reset all of its settings to default before applying your configuration.
  • boolean persist
    • When the robot turns on and off, the settings on your motor controller will go back to what was last saved to memory. If you set persist to true, it will save the settings to the motor controller’s memory. Otherwise, this configuration will be switched back to what was last on the memory after a power cycle. Saving the configuration to memory will allow the settings to persist through power cycles.

In most cases, reset will be set to true. It’s better in most cases to reset to default settings before applying your own so that you know exactly what settings are on your motor controller during debugging. persist will also be true in the final version of your code. However, during testing you should set persist to false to avoid wearing out the memory with too many saves.

    SparkFlex motor = new SparkFlex();

    SparkFlexConfig sparkFlexConfig = new SparkFlexConfig();

    sparkFlexConfig

     .idleMode(IdleMode.kBrake)

     .smartCurrentLimit(60);

    motor.configure(sparkFlexConfig, true, true);
