Lambdas and Suppliers

Every once in a while in programming, you'll see lambdas. These are anonymous functions, or functions with no name. They look like this:

() -> {}

Here, parameters go in the parenthesis and the function body in the curly braces, as with a normal function. Ex:

    (x,y) -> {

      System.out.println(x);

      System.out.println(y);

      return x+y;

    }

Lambdas can be super useful because they can be passed as arguments into a method (kind of like passing a method to a method)! You actually used lambdas in Exercise Four when setting the robot’s movement!

    double speed = 5;

    Scratchbot.sb_setRobotMovement(

      () -> {return -Scratchbot.sb_getControllerLeftY()},

      ()-> {return -Scratchbot.sb_getControllerLeftX()}

      );

But why does this matter? Well, let's see what would happen if we store the values of these methods in variables first and then pass those as arguments:

    double speed = 5;

    double arg1 = -Scratchbot.sb_getControllerLeftY();

    double arg2 = -Scratchbot.sb_getControllerLeftX();

    Scratchbot.sb_setRobotMovement(arg1,arg2);

If the value of arg1 and arg2 both turned out to be 0.5. then the method call would really be:

    Scratchbot.sb_setRobotMovement(0.5,0.5);

This is because the method is only called when we initialize the variable’s value, so the variable will be assigned to whatever the value was at that particular time. If the joystick moves after this singular call, it won’t update the value of the variable since we’re never re-evaluating the value.

However, if we input a lambda for each argument, that value will be updated EVERY time the method runs allowing the value to change over time.

In order to make a method that takes in a lambda expression as an argument you can use a supplier. This method would have looked something like:

    public static void Scratchbot.sb_setRobotMovement(Supplier xVelocity, ){...}

Suppliers can be of any type! Here, -Scratchbot.sb_getControllerLeftX() returns a double so we put Double inside the <> to signal that no other datatype should be accepted.

NOTE: Suppliers only take in non-primitive data types. This means inputting doubles, floats, ints, and chars will throw an error. In order to avoid this use what’s called a wrapper class. These classes are (mostly) the same as their primitive counterparts but they can be used with Suppliers. The following are the equivalent wrapper classes for each primitive datatype:

double -> Double

int -> Integer

char -> Character

float -> Float

In order to get the value of a Supplier use .get():

    Supplier exampleIntSupplier = () -> {return 5;};

    int exampleInt = examplIntSupplier.get();

    System.out. println(exampleInt); //would print 5
