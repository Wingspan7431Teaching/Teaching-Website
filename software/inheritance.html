Inheritance in FRC

Inheritance is HEAVILY used in FRC programming. One example of this lies in the structure of the Robot.java file. This file holds a class, Robot, that is a child of TimedRobot. Robot.java file. This file holds a class, Robot, that is a child of TimedRobot. TimedRobot is a Class that offers a wide variety of methods that Robot overrides. This class is protected and is intended to be inherited from TimedRobot.

The most common methods that are overridden in the Robot class from TimedRobot are:

  • robotInit()
  • robotPeriodic()
  • autonomousInit()
  • autonomousPeriodic()
  • autonomousExit()
  • teleopInit()
  • teleopPeriodic()
  • teleopExit()

The reason these need to be overridden is that it allows the library to call the methods at the right time for us. For example, teleopInit() will be run at the start of the teleoperated period of the game regardless of what code we put in the body of the method.

  • robotInit()
    • Runs when the robot first turns on
  • robotPeriodic()
    • Runs once every cycle, regardless of what section of the game it is
  • autonomousInit()
    • Runs at the start of the autonomous period
  • autonomousPeriodic()
    • Runs once every cycle during the autonomous period
  • autonomousExit()
    • Runs at the end of the autonomous period
  • teleopInit()
    • Runs at the start of the teleoperated period
  • teleopPeriodic()
    • Runs once every cycle during the teleoperated period
  • teleopExit()
    • Runs at the end of the teleoperated period
