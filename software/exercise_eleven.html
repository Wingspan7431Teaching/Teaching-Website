Exercise Eleven

For this exercise you’ll be creating a command for Drivetrain. In order to keep the focus on Command-Based Programming, you’ll be using Scratchbot methods here rather than calling methods that are in the Drivetrain class.

Navigate to the Drivetrain folder.

Next, navigate to the MockMoveRobot file. It should have the template code from below:

    public class MockMoveRobot extends Command {

      pose2d goalPose;

      Supplier<Boolean> isPressedA;

      public MockMoveRobot(pose2d goalPose, Supplier<Boolean> isPressedA) {

        // add Subsystem Requirements

      }

      @Override

      public void initialize() {}

      @Override

      public void execute() {}

      @Override

      public void end(boolean interrupted) {}

      @Override

      public boolean isFinished() {

        return false;

      }

    }

Task: Complete the MockMoveRobot command so that it moves the robot when the A button is hit.

Specifications:

  • The constructor should initialize the class variables and add the subsystem requirements.
  • The parameter goalPose is the position the robot should move to.
    • Note: You can assume that the starting position is the origin.
  • isPressedA is a boolean supplier that holds the current value of the A button.
  • The initialize() or execute() method (it’s up to you to determine which) should check the value of the isPressedA and call the necessary Scratchbot method when it's pressed.
  • The end() and isFinished() methods do not need to be altered.
  • Within the teleopInit() method of the RobotState.java file, create an instance of MockMoveRobot.
  • Set the MockMoveRobot instance to be the default command for the Drivetrain subsystem.
  • Make sure CommandScheduler.getInstance().run() is in the robotPeriodic() method so that your code will run.

  • Suppliers.
  • pose2d Class.
  • Command-Based Programming, Commands, and Default Commands.
