@Override

In the lesson about inheritance you learned that subclasses inherit the same attributes and methods that their parent class has. Then, in the lesson on polymorphism, you saw how you can change (or override) a method from a superclass by redefining it in a subclass. This is called overriding.

A good practice is to write “@Override” above any method that’s been overridden. This signals to the computer and other programmers that it’s an overridden method. If @Override is used and the method is not overridden, the computer will throw an error.
