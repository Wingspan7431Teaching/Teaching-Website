Is This an Action or Hardware?

Now, what code goes into the command and what code goes into the subsystem?

We’ve discussed that commands should handle input and the action while subsystems should handle setting the hardware. However the line between that can get blurry when you're coding. Different teams draw this line differently; for Wingspan, we say the following:

Commands should get input from the driver or sensors and convert it into a form that the subsystems can use without having to have different methods for every single command.

For example, say you’re making the subsystem and commands for the arm and you have the following commands:

  • Raise the arm to a certain height when bot is in a specific region of the field
  • Raise to max height when the driver hits the back bumper button
  • Raise the arm up and down based on the y-axis of a joystick

These commands should all call the same method, setHeight(double height), in the subsystem.

An example of what NOT to do is making unique methods in the subsystem such as:

  • setHeightWithRegion(Pose2d robotPosition)
  • setHeightWithBumper(boolean isBumperPressed)
  • setHeightWithJoystick(double joystickValue)
