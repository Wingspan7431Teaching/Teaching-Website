Running Commands

We’ve already seen how .setDefaultCommands() can be used to set a default command for the subsystem to run when no other commands are being run like an “idle behavior.” In order to get a fully functioning robot, we also need to have other actions based on input. This is what we use triggers for.

Triggers are instances of the Trigger class. A trigger runs based on the value of a boolean value— which could be something like a button or a sensor. For example, if we want to turn an LED on when the A button is pressed, the A button being pressed would be our trigger and turning the LED on would be the command.

You can make a trigger using the constructor for the Trigger class. The constructor for this class takes one parameter: a Supplier<Boolean> for whatever condition you want to be the trigger. Then, you can take that trigger and use the .onTrue() method on it. The parameter to onTrue() is the command you want to run. For the MockMoveRobot example the code would look something like this:

    XboxController controller = new XboxController(0);

    MockMoveRobot moveInstance = new MockMoveRobot(Pose2D goalPoses);

    Trigger aButtonTrigger = new Trigger(() -> {return

      controller.getAButtonPressed()});

    aButtonTrigger.onTrue(moveInstance);

Notice how the MockMoveRobot constructor doesn’t need the isPressedA parameter anymore; the trigger handles running the command for you. There’s also a method for .onFalse() if you’d like to run a command when a value is false rather than true.

Another way to do this is using the JoystickButton class which is a subclass of Trigger. This class works the same as Trigger, but it’s made specifically for controller input and doesn’t use suppliers. The constructor is: JoystickButton(XboxController controller, int buttonValue), where buttonValue is a button value in the form XboxController.Button.kY.value. For example:

  • XboxController.Button.kLeftBumper.value
    • The left bumper button’s value
  • XboxController.Button.kB.value
    • The B button’s value

Let’s look at the same example from earlier but with JoystickButton instead of Trigger:

    XboxController controller = new XboxController(0);

    MockMoveRobot MockMoveRobot = new MockMoveRobot();

    JoystickButton aButtonTrigger = new JoystickButton(controller,

      XboxController.Button.kA.value);

    aButtonTrigger.onTrue(MockMoveRobot);

There’s other ways to schedule methods, but you can complete most tasks with just these ones.

Note: Something like a joystick input would typically be a default command because the value ranges from -1 to 1, rather than being a boolean value like whether a button is pressed.
