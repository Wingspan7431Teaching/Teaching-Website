JavaDoc

When working on a team (or really any big project), it’s important to write documentation for your code. JavaDoc is typically used to create Java API documentation. Let’s look at an example of a JavaDoc comment:

    /**

    * Computes the sum of two numbers

    *

    * @author John Programming

    * @param a The first number to add

    * @param b The second number to add

    * @returns The sum of a and b

    */

    public int sum(int a, int b){

      return a+b;

    }

Notice that a JavaDoc comment is created by using /** content */ while a normal multi-line comment is /* content */.

The first part is a description of the method. Then, there are tags. The @author tag specifies who wrote the method. The @param tag specifies what a specific parameter is. @returns specifies what the method returns.

Try adding JavaDoc comments to a method, then hover over the name of the method. Notice how the documentation is displayed.

There are other additional tags that you can find online as well. Note that there are no required tags. For example, a JavaDoc comment might look like:

    /**

    * Computes the sum of two numbers

    */

    public int sum(int a, int b){

      return a+b;

    }

Just make sure you include enough information for anyone that isn’t familiar with your method to understand what it is and how to use it.

You can also write JavaDoc comments for variables. This can be useful to specify units or what a variable is used for.

    /** The default drivespeed that is set for the drivetrain in meters/second*/

    public static final double defaultDriveSpeed = 0.5;
