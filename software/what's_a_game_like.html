What's a Game Like? (For Programming)

In an FRC game there’s two alliances, the red alliance and the blue alliance. Each alliance has three teams for a total of six robots on the field.

At the start of a game, there’s 15 seconds called autonomous. During this period, the robots must function without any human input or interruption. In autonomous, most scoring options will give you more points. This is where teams with super good programming get the chance to really test their skills.

After autonomous the game enters the 2 minute tele-operated period, or “teleop.” Teleop is when drivers control the robot. This is usually a bit easier to code because it’s simply converting user-inputs to outputs instead of using sensors for inputs. However, for efficiency in teleop, programmers must create code that will help the driver score. This might look like the robot automatically aligning itself to score, or having preset positions for a scoring-arm so the driver doesn’t have to painstakingly raise it to the right height.

Feel free to look up some past games to get a feel of what you’ll be working on!
