Exercise Six

Now that you have a good idea of what OOP is and how it works, it’s time to use some WPILib classes! WPILib or WPI Robotics Library is a massive library of classes and methods that you can import and use in your code. WPILib is what FRC teams use to code their robots! If you want to look at their website you can find it here. After you complete these lessons, the WPILib docs and WPILib Java API docs will be very useful for more in depth information or complicated topics.

One class WPILib has is the XboxController class. This class is very similar to what was explained in the What's OOP? lesson. It allows you to create a controller and get input from it super easily!

The goal of this exercise is to use this class to redo Exercise Four using the XboxController class methods instead of Scratchbot.sb_getControllerLeftX() and Scratchbot.sb_getControllerLeftY(). You may reuse the code you wrote in that exercise as a base.

Some additional information you might find useful is:

  • Two methods for instances of the XboxController class are getLeftX() and getLeftY()
  • You can import WPILib classes by doing the following:
    1. Create an object of the class
    2. Hover over the red-line that will appear
    3. Use “Ctrl + .” to see the quick fix options
    4. Import the class
  • WPILib classes all have documentation. This allows you to hover over a class or a method to view information about it. (You may want to use this to find information about the constructor)
  • Use either 0 or 1 for the port number. This can be configured in Driver Station
