Exercise Thirteen

For this exercise, you’ll be writing all of the code for a motor subsystem & command without the use of any Scratchbot methods. You should make:

  • A motor subsystem file
    • The motor should be configured in this file
    • The motor should have its speed set in this file
  • A constants file for the motor subsystem
  • A command to spin the motor based on input from a joystick
  • Code in the Robot.java file to create the controller, and make the command a default command

Notes:

  • The command should call a method from the subsystem to spin the motor.
  • The motor’s speed should depend on the joystick’s Y axis (a value of 1 would mean the joystick is being pulled towards the user and the motor should spin in reverse).
  • Commands should not directly interact with the motor, they should call subsystem methods to interact with the motor.
  • Feel free to reference code you've already made while working on this. Memorizing code or syntax is never required nor expected; although, it tends to happen naturally overtime.
