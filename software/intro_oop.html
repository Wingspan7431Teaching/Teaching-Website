What's OOP?

In the upcoming lessons, you'll learn more about Classes and Objects on W3Schools. This lesson aims to give you the relevant, robotics-specific, context


So far you’ve been coding using what’s called procedural programming. This means that you write steps (procedures) that go in order and use variables that each store a single piece of information. OOP stands for Object-Oriented Programming which basically means that the code is organized into objects that contain more data, and methods that manipulate that data.

OOP uses 2 main concepts:

  • Classes: Templates that say what an object will be. It says what data this object will hold, what actions it will have, and how to make one
  • Objects: A specific instance of the class.

Methods are the actions that can be used on the objects. Attributes are variables/data in the class.

==✧==

For example, if you have two controllers that both need to:

  • Know the computer port it's plugged into
  • Get the position of the left joystick
  • Check if the X button is being pressed

Then a Controller class could be made that offers a template to make both of these controllers.

This makes it where there’s no copy and pasting code to handle each controller. The controller class might contain:

  • computerPort
    • A variable that specifies what port of the computer this controller is in.
  • getLeftX()
    • Method that defines how to get the left joystick’s x coordinate for a specific controller object
  • getLeftY()
    • Method that defines how to get the left joystick’s y coordinate for a specific controller object
  • Other Controller Methods

If we have an object of this Controller class—let's call it controllerA—then it might have:

  • computerPort = 1
    • ControllerA's specific computer port value
  • getLeftX()
    • Method that will return controllerA's left joystick's x coordinate.
  • getLeftY()
    • Method that will return controllerA's left joystick's y coordinate.
  • Other Instance Methods

==✧==

Quick Definition, Instance: Oftentimes in programming you’ll hear the term “instance” used to refer to an object of a class. For example, controllerA is an instance of the Controller class.
