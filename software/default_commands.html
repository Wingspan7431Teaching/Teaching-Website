Default Commands

Once you have a subsystem and commands for it, you need a way to tell the CommandScheduler to run a command. One of the simplest ways to run a command is by setting it as the default command for the subsystem.

Default commands are like an idle behavior; the command runs when no other commands are scheduled. When another command is scheduled, it interrupts the default command. After the new command is done running, the default command will resume. The following syntax is used to set a default command for a subsystem:

ExampleSubsytem.setDefaultCommand(exampleCommandInstance);
