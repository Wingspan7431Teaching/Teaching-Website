Loops in FRC

A lot of the code written in FRC runs in cycles. This allows the code to respond to new input every cycle. Because of the short nature of these cycles, loops can easily go over the allotted time and cause errors.

You’ll learn more about this later, but for now just keep these things in mind:

  • One cycle is 20ms by default.
  • If your code takes longer than 20ms, it can cause bugs or lag in the robot’s behavior.
  • While loops should be avoided because they can easily go over the 20ms limit and freeze the robot.
  • For loops are okay, but they should still be fast and finish within that cycle.
